## Introduction

Maniac est une bibliothèque de composant [Vue](https://vuejs.org/) dogmatique prête à être utilisée pour des projets [Vite](https://vitejs.dev/).
Elle est conçue pour être utilisée comme un point de départ pour vos propres projets.

## Our principles

En tant que développeur front-end,
le problème des bibliothèques comme [Bootstrap](https://getbootstrap.com/) ou [Vuetify](https://vuetifyjs.com/en/) est que l'on délègue une partie du socle de notre application
à une librairie externe de composant qu'on ne peut pas toujours manipuler et adapter comme on le souhaite,
rendant parfois difficile la customisation de ces composants, et très compliqué de se séparer de ces librairies une fois fortement installé dans un projet.
De plus, certaines librairies apportent des fonctionnalités qui vont impacter l'ensemble de votre projet pouvant fragiliser la maintenabilité de votre projet.

Contrairement à ces autres bibliothèques, Maniac vient s'installé au coeur de votre projet pour permettre à n'importe qui de customiser entièrement ces composants.
Elle est composée de plusieurs composants, qui sont tous développés avec amour pour le développement front-end, en suivant les bonnes pratiques avec l'objectif de travailler en équipe et récupérer un socle de composants stables.

Dans ce but, les composants disponibles répondent à de nombreux critères de qualité, de maintenabilité et facile à manipuler pour des développeurs de tout niveaux.
Maniac offre une totale transparence sur le développement de ses composants pour garantir une qualité de code stable et une maintenabilité optimale tout au long de votre projet.
De plus, les [bonnes pratiques](?path=/docs/guidelines-best-practices--page) présentées peuvent être applicable à tous les projets.

## Contribute

Si vous souhaitez contribuer à ce projet, il est recommandé de prendre en compte ces quelques règles afin de respecter le code et la qualité du projet :

- Avant de commencer à développer une fonctionnalité, il est fortement conseillé d'en discuter via une issue [Github](https://github.com/kevbesset/maniac) afin d'éviter des doublons ou d'apporter des fonctionnalités contradictoire à la phylosophie de Maniac.
- Ensuite, s'il s'agit d'une suggestion dans le code, il est recommandé de lire les [bonnes pratiques](?path=/docs/guidelines-best-practices--page) pour éviter les erreurs de développement et de qualité.
  Pour rappel, lorsque vous modifiez ou créez un nouveau composant, il faut s'assurer que la ligne de commande, la story dans le design system ainsi que le composant lui-même fonctionne correctement.
- Puis vous pouvez ensuite créer une pull request qui sera reviewée et ajoutée au projet après validation.

## Upgrade guide

Puisque les composants sont ajoutés directement dans votre projet, il devient ensuite difficile de faire des mises à jour si vous venez à les modifier fortement.
Dans le cas où vous souhaitez quand même mettre à jour les composants, il est recommandé d'utiliser Git pour déterminer les changements et gérer les différences.

Si vous n'avez pas modifier les composants, vous pouvez sereinement écraser avec les composants à jour.

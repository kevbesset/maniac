## Overview

Étant donné que les composants viennent s'intégrer directement dans le code source de votre application,
il est important que ceux-ci suivent une convention de code stricte et pertinente.

Dans Maniac, nous avons fait le choix de construire nos composants sur des valeurs centrales :

- La souplesse et la maintenabilité du code.
  En effet, ces composants doivent pouvoir être utilisés sur votre plateforme sans problèmes et être potentiellement adapter par votre équipe sans trop de difficultés.
- Durant la conception des composants, ceux-ci doivent réagir rapidement et de manière fluide pour ne pas penser à des problèmes de performance.
- Tous les composants sont pensé pour être utilisés dans un contexte d'accessibilité.
  Nous ne garantissons pas de niveau particulier d'accessibilité comme nous n'apportons que très peu de contexte à nos composants. Cependant, le service minimum est assuré.
- Les composants sont réfléchi pour une expérience utilisateur fluide, avec des animations naturelles de transition et tous les effets dû aux actions utilisateurs représentés.

Bien sûr, ces règles sont appliqués à tous les composants, mais nous vous recommandons de les respecter dans votre application pour une meilleure harmonie au sein de votre projet.
